GitHub Webhook Monitor

A lightweight FastAPI service for receiving, validating, and inspecting GitHub webhook deliveries.

What it does

  • Receives GitHub webhook deliveries via POST /webhook/github.
  • Validates signatures using HMAC SHA256 and X-Hub-Signature-256.
  • Rejects invalid deliveries with 401 Unauthorized.
  • Stores a capped in-memory list of recent events for inspection.
  • Exposes GET /health and GET /events endpoints.

Quick start

git clone https://github.com/Franklindot04/github-webhook-monitor.git
cd github-webhook-monitor
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# Edit .env with your values
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

Then open http://127.0.0.1:8000/docs for interactive API docs.

GitHub webhook setup

  • Payload URL: https://your-domain.com/webhook/github
  • Content type: application/json
  • Secret: same as WEBHOOK_SECRET in your .env
  • Events: start with Pull requests or Just the push event.

For local testing, expose your dev server with a tunnel (e.g. ngrok) and use the public URL as the payload URL.
