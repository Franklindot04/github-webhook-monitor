# Contributing

Thanks for your interest in contributing to GitHub Webhook Monitor.

## How to contribute
- Fork the repository.
- Create a feature branch from `main`.
- Keep changes focused and small.
- Test your changes locally before opening a pull request.
- Update documentation when behavior changes.

## Development notes
- The project is built with FastAPI.
- Environment variables are configured through `.env`.
- The webhook receiver validates GitHub webhook signatures.

## Pull requests
Please include:
- A clear summary of the change.
- Why the change was made.
- Any local testing steps used to verify it.
